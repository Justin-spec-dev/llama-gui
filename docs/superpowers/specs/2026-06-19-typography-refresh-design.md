# llama-gui 字体与密度优化设计

## 目标

在不改变工作台布局、交互和功能的前提下，提高中文界面的可读性与视觉一致性。最终版本继续支持浅色、深色和跟随系统主题。

## 字体体系

- 全局字体优先使用 `Microsoft YaHei UI`，依次回退到 `Segoe UI Variable`、`Segoe UI` 和系统无衬线字体。
- 正文基础字号为 11.5 pt。
- 输入框、下拉框、数字框和按钮使用 11.5–12 pt，并提高最小高度。
- 导航主标签、分区标题和关键状态使用 13–15 pt 的半粗体。
- 说明文字使用 10.5 pt 与弱化颜色，保持信息层级。
- 日志区域使用 `Cascadia Mono`、`Consolas` 等宽字体，字号 10.5–11 pt。

## 密度与布局

- 输入类控件最小高度调整到约 30 px，按钮最小高度约 32 px。
- 导航条目增加垂直留白，保证双行文字不拥挤。
- 顶部命令栏保持单行布局，适度增加内边距，不扩大结构占用。
- 现有最小窗口尺寸 1024×700 保持不变；所有页面仍可滚动。

## 实现边界

- 修改应用全局字体和主题 QSS 中的字体、字号、字重、控件高度与间距。
- 必要时为导航和日志控件增加对象级字体样式。
- 不改变颜色方案、页面结构、配置字段、服务器生命周期或预设逻辑。

## 验收标准

- 中文与英文混排协调，标题、正文、说明和日志层级清楚。
- 1024×700 下按钮和输入框不裁切，导航双行文字完整显示。
- 浅色和深色主题均正常。
- 自动化测试通过，重新生成的单文件 EXE 可独立启动。
